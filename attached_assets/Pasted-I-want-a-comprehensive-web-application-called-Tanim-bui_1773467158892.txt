I want a comprehensive web application called Tanim, built exclusively using PHP and Blade templates within the Laravel 11 framework and Tailwind CSS, designed to manage an agricultural supply chain. The system must be powered by a MySQL database compatible with XAMPP and MySQL Workbench, following a 2nd Normal Form (2NF) architecture, and provided with full migration scripts and seeders. The application must feature a multi-role user management module for Buyers, Farmers, and Admins, supporting profile photo uploads, an account status toggle (Active/Inactive), and a secure authentication system that requires email verification before granting access to protected PHP/Blade routes. For the core marketplace functionality, implement a product management system using Blade-driven datatables that allow for multiple photo uploads per item and a feature to import product listings directly from an Excel worksheet.

The platform must include a persistent shopping cart and a checkout process that supports both Cash on Delivery (COD) and online payments, with backend PHP logic to automatically deduct stock levels upon order confirmation and notify farmers when quantities drop below a set threshold. To facilitate customer feedback, incorporate a review and rating system where buyers can post and update their comments on purchased goods via Blade forms. Advanced search capabilities should be powered by Laravel Scout, complemented by detailed filters for product categories—such as Grains, Vegetables, and Fruits—as well as price range sliders.

Finally, the application must provide an administrative dashboard featuring Chart.js visualizations integrated into Blade views, specifically a yearly sales bar chart and a pie chart illustrating the percentage of total sales contributed by each product. For the order fulfillment phase, the system must allow administrators to update transaction statuses and, upon completion, use PHP to automatically generate a detailed PDF receipt via DomPDF to be sent to the customer through email. This build should ensure all functional requirements, including procurement, expense monitoring, and delivery tracking, are seamlessly integrated and ready for presentation.
